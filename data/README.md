# Data

This folder contains all the datasets you need to replicate the analysis. Some of them are generated automatically by the data pipeline, others are compiled manually from public sources, and a few are small files we built directly for the model. Below we explain what each file is, where it comes from, and why we made the choices we did.

---

## What's in here

The main output of the data pipeline is `historical_data.csv` — a complete hourly dataset of the Spanish electricity system from 2020 to 2024 (around 44,000 rows). This is the file the Julia model reads directly for calibration. All physical quantities are expressed in GW or GWh, and all prices are in €/GWh.

The rest of the files feed directly into the Monte Carlo simulation:

- `projection_deltas_data.csv` contains the growth estimates used to project historical patterns to 2030. For each key variable (installed capacity, demand, fuel costs...) we compiled estimates from several forecasting institutions (PNIEC, Aurora Energy Research, Red Eléctrica, IEA...) and stored them as relative deltas with respect to the baseline year. In each Monte Carlo iteration, a delta is randomly drawn from this distribution.

- `scenario_params.csv` and `technical_params.csv` hold parameters that remain fixed across all simulations but differ between scenarios or model components — things like demand elasticities, hydro availability adjustments, or storage efficiencies.

- `technology_data.csv` contains one row per generation technology and includes everything the model needs to compute costs and emissions: efficiencies, variable and fixed O&M costs, and CO₂ emission intensities. 
The cost metrics (variable and fixed O&M) and emission intensities in this file are largely derived from the US National Renewable Energy Laboratory (NREL) Annual Technology Baseline (ATB). While NREL is a US-based institution, its ATB dataset is widely considered a gold standard in global energy systems modeling. We use this dataset because core power generation technologies (like solar PV modules, wind turbines, and gas turbines) are traded in highly globalized supply chains, making their relative capital and operational costs internationally comparable. Using NREL ATB ensures our model relies on a transparent, standardized, and peer-reviewed baseline for the technical and economic performance of these assets.

**NREL Data Transformation Methodology:**
To adapt the raw NREL ATB metrics for the Julia model, the following mathematical conversions were applied:
*   **Efficiency:** Derived from NREL's Heat Rates using the physical conversion `Efficiency = 3.412 / Heat Rate [MMBtu/MWh]`.
*   **Variable O&M (`var_om_eur_gwh`):** NREL reports these costs in `$/MWh`. Assuming an approximate long-term 1:1 EUR/USD parity, this translates to `€/MWh`. It is then multiplied by 1000 to match the model's energy scale of `€/GWh`.
*   **Fixed O&M (`fixed_om_Meur_gwy`):** NREL reports fixed costs in `$/kW-yr`. Mathematically, `1 $/kW-yr` is exactly equivalent to `1 Million $ / GW-yr`. Assuming 1:1 EUR/USD parity, the raw NREL values are directly adopted as `Millions of € / GW-yr`.
*   **Direct Emissions (`direct_e_tco2_gwh`):** Based on standard IPCC/NREL carbon intensities for fossil fuels, scaled to tons of CO₂ per GWh (e.g., ~360 tCO₂/GWh for Combined Cycle Gas).

There is also an `other/` subfolder with a few additional files explained further below.

## How `historical_data.csv` is built

The main dataset is generated by running `scripts/Codigo_historical_data.py`. The script pulls data from two APIs and a set of fuel cost sources, processes everything, and writes the final CSV to this folder. It typically takes 60–90 minutes to run, depending on API response times.

To run it you will need valid API tokens for:
- **ESIOS** (Red Eléctrica de España) — request one at [api.esios.ree.es](https://api.esios.ree.es)
- **ENTSO-E** — request one at [transparency.entsoe.eu](https://transparency.entsoe.eu)

Once you have them, set them at the top of the script and run it from the root of the repository:

```bash
python scripts/Codigo_historical_data.py
```
.
### What comes from ESIOS

ESIOS is the data portal of Red Eléctrica de España and it is the main source for the Spanish electricity system data. The script downloads:

- Hourly generation broken down by technology (coal, combined cycle, nuclear, wind, solar PV, solar thermal, hydro, cogeneration, and more)
- Sectoral electricity demand (residential, commercial, industrial) at different voltage levels
- Hourly spot prices from the OMIE day-ahead market
- Net interconnection flows with France, Portugal and Morocco
- Monthly installed capacity for all generation technologies
- Self-reported available nuclear capacity

### What comes from ENTSO-E

ENTSO-E's transparency platform provides more granular hydro data than ESIOS. We use it specifically for:

- Run-of-river hydroelectric generation
- Reservoir (conventional) hydro generation
- Pumped storage generation and consumption

This disaggregation matters for the model because run-of-river, conventional hydro and pumped storage behave very differently from a dispatch perspective.

### Fuel and CO₂ costs

Fuel costs are the trickiest part of the dataset because there is no single API that provides all of them cleanly. Here is what we use and why:

**Natural Gas — MIBGAS (Iberian Gas Market)**

The script downloads gas prices directly from the official MIBGAS historical data files. Rather than using the more common Dutch TTF benchmark, we use MIBGAS because it reflects the actual price faced by Spanish combined cycle gas turbines — the Iberian gas market can decouple from Northern European prices due to pipeline constraints and the LNG import infrastructure in Spain and Portugal.

**Coal — API2 Rotterdam (Yahoo Finance, ticker `MTF=F`)**

API2 Rotterdam is the standard benchmark for thermal coal imported into Northwest Europe and the one most relevant for Spanish coal plants. The raw data comes in USD/ton, so we convert it to EUR/MWh using the daily EUR/USD exchange rate (also from Yahoo Finance, ticker `EURUSD=X`) and a thermal conversion factor of 8.14 MWh per metric ton.

**Diesel / Gasoil — NYMEX Heating Oil (Yahoo Finance, ticker `HO=F`)**

The European ICE Gasoil index is the more natural reference for diesel in Spain, but high-frequency historical data is not freely accessible. The NYMEX Heating Oil futures contract is a very close proxy — both track middle distillates and are highly correlated. Raw prices are in USD/Gallon and are converted to EUR/MWh using the EUR/USD rate and a factor of 0.0406 MWh per US gallon.

**EU ETS Carbon Price — Local Excel file (fallback)**

High-frequency carbon market data is restricted by exchanges like ICE. For our purposes — modeling the hourly merit order — monthly averaged ETS prices provide a sufficient economic signal (the penalty per emitted ton of CO₂) without introducing artificial intra-day noise. The script tries to pull this data automatically from Yahoo Finance (`^ICEEUA`), but if that fails it falls back to the manually compiled file `data/other/Coal_Diesel_ETS_Monthly_Costs.xlsx`, which aggregates official monthly averages from sources such as Ember Climate and ICE EUA reports.

**Uranium — Local Excel file (fallback)**

Uranium does not trade on a liquid daily spot market. Most transactions happen through long-term bilateral contracts, and the main price references (UxC, Cameco) publish weekly or monthly surveys. The script tries to pull uranium futures from Yahoo Finance (`UX=F`), but since that ticker covers a short and unreliable historical window, it almost always falls back to the manually compiled file `data/other/Uranium_Annual_Prices2020-2024.xlsx`. Prices are in USD/lb and are converted to EUR/MWh using a nuclear fuel energy density factor of 30.211 MWh per pound of U₃O₈. Annual averages are appropriate here because nuclear operates as a baseload technology — its hourly dispatch is not sensitive to short-term spot price movements.

## The `other/` subfolder

This subfolder holds a few files that are either manual compilations or intermediate outputs:

- `NaturalGas_Daily_Prices2020-2025.xlsx` — a local backup of MIBGAS daily gas prices, used as fallback if the MIBGAS API is unreachable.
- `Coal_Diesel_ETS_Monthly_Costs.xlsx` — monthly coal, diesel and EU ETS prices, compiled manually. Used as fallback for ETS (and coal/diesel if yfinance is unavailable).
- `Uranium_Annual_Prices2020-2024.xlsx` — annual uranium prices in USD/lb, from UxC and Cameco market reports. Primary source for uranium costs.
- `fixed_data_euros.csv` — fixed O&M cost estimates by technology from the literature, used during model calibration.

> These files do not need to be updated to replicate the paper. They are kept in the repository as permanent fallbacks and as a record of the manually sourced data.

## Missing data

Raw API data inevitably has gaps — market holidays, server issues, indicator outages. The pipeline handles this with a three-step imputation applied in order:

1. **Short gaps (1 hour)** are filled by copying the value from the previous hour.
2. **Longer blocks** are filled by copying values from exactly 24 hours earlier, preserving the natural daily rhythm of demand and generation. Spot prices are excluded from this step and handled separately with linear interpolation, since yesterday's price is not a good proxy for today's.
3. **Anything remaining** is filled by linear interpolation, with a final backward fill to handle edge cases at the very start of the series.

After imputation, all physical quantities (generation, demand, capacity) are clipped to zero — negative values have no physical meaning here.
